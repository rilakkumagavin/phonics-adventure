# A 字母圖片資產規格

## 目的

本文件定義 A 字母課程第一批圖片資產的風格、尺寸、構圖、檔名、路徑、alt 文字、生成提示詞與驗收標準。

本階段只建立規格與生成清單，不生成圖片、不下載圖片、不替換課程頁 placeholder。

## 適用範圍

適用於 A 課程中的：

- 核心單字圖：apple、ant、alligator。
- 句子情境圖：An apple.、I see an ant.、The alligator is big.
- 活動選項圖：聽音選圖、看字選圖、複習活動中的正確選項與干擾選項。
- 複習圖：看圖選字母、聽聲音選圖、判斷開頭音、新情境複習。

UI 輔助圖如 A 字母關卡封面、A 課程主視覺、完成徽章可延後，不列入本批圖片。

## 採用視覺風格

本專案 A 課程圖片採用：

> 柔和 3D 卡通教育插畫，圓潤造型、明亮自然色、乾淨背景、主體置中、表情友善、適合兒童學習網站。

選擇理由：

- 現有網站使用明亮色彩、圓角卡片與清楚觸控區塊，柔和 3D 卡通風格能自然搭配。
- A 課程目標使用者為英語初學者與國小低年級至中年級學生，主體需要快速辨識且不造成壓力。
- apple、ant、alligator 需要在活動與複習中重用，圓潤 3D 造型比寫實圖更容易保持友善與一致。

## 尺寸與比例

### 核心單字卡圖片

- 比例：1:1
- 來源尺寸：1024x1024
- 格式：WebP
- 用途：課程頁單字卡、聽音選圖、閱讀活動、複習活動。
- 背景：簡單純色、柔和漸層或極簡場景。

### 句子情境圖

- 比例：4:3
- 來源尺寸：1200x900
- 格式：WebP
- 用途：短句跟讀、句子配圖、後續複習。
- 背景：允許簡單場景，但場景只服務句意。

### 課程封面或主視覺

- 比例：16:9 或 3:2
- 建議來源尺寸：1600x900 或 1500x1000
- 狀態：可延後。
- 理由：目前 A 課程頁已用字母與內容卡片承接，不需要在本批圖片加入主視覺。

## 構圖原則

- 主體置中，完整顯示，四周保留足夠邊界。
- 不裁切主體，不讓重要部位貼近邊緣。
- 核心單字圖只呈現一個主要目標物件。
- 活動圖片需要可在小卡片中辨識，不依賴細節。
- 句子情境圖可以有簡單場景，但不得加入會混淆目標單字的多餘物件。
- 不在圖片中顯示英文字母、英文單字、中文、數字標籤或教學文字。

## 背景原則

### 核心單字圖

- 使用簡單純色、柔和漸層或極簡背景。
- 主體與背景要有足夠對比。
- 不使用複雜教室、街景、森林等大量背景。
- 不使用大量裝飾。

### 句子情境圖

- 允許桌面、野餐墊、簡單草地、河岸等情境。
- 場景只服務句意，不增加無關人物與物件。
- 不讓學習者誤判目標單字。

### 活動圖片

- 正確圖與干擾圖要有接近的視覺複雜度、光線、背景質感與精緻度。
- 干擾選項不可因背景明顯不同、細節較少或畫風較差而洩漏答案。
- 不可只有正確答案使用精緻插畫，其他選項使用簡圖。

## 色彩原則

- 使用明亮自然色，避免高飽和刺眼色。
- apple 固定紅色，帶一片綠葉。
- ant 固定棕色或深紅棕色，圓潤卡通造型。
- alligator 固定綠色，友善表情，不使用暗沉恐怖色。
- 背景可使用接近現有網站色彩的柔和綠、淺藍、白色或暖色。
- 不使用品牌色、商標色塊或真實商品包裝。

## 角色與物件一致性

同一單字若出現在多張圖片中，需保持：

- 外型一致。
- 主色一致。
- 比例一致。
- 表情一致。
- 畫風一致。
- 不要每次換成完全不同物件或不同畫風。

固定規則：

- apple：紅色、圓潤、單一蘋果、一片綠葉。
- ant：棕色或深紅棕色、一隻、搬著一小塊食物、友善、不寫實。
- alligator：綠色、圓潤、友善、嘴巴微張、不具威脅感。

若後續要使用同一位兒童角色，需另開角色資產規格；本階段不建立完整角色系統。

## 禁止內容

所有圖片禁止出現：

- 英文字母或單字。
- 中文。
- 水印。
- 商標。
- 品牌包裝。
- 恐怖牙齒。
- 攻擊畫面。
- 傷害畫面。
- 過度寫實昆蟲。
- 複雜背景。
- 不必要人物。
- 不相關道具。
- 模糊主體。
- 裁切主體。
- 多個競爭焦點。

## 目錄與檔名規則

本階段規劃採用 `public/assets`。

建議目錄：

```text
public/
└─ assets/
   └─ images/
      ├─ courses/
      │  └─ a/
      │     ├─ a-apple-core.webp
      │     ├─ a-ant-core.webp
      │     ├─ a-alligator-core.webp
      │     ├─ a-sentence-an-apple.webp
      │     ├─ a-sentence-see-ant.webp
      │     └─ a-sentence-big-alligator.webp
      └─ shared/
         ├─ shared-ball-core.webp
         ├─ shared-cat-core.webp
         └─ shared-dog-core.webp
```

選擇 `public/assets` 的理由：

- A 課程資料目前以 URL 字串參照圖片，如 `/assets/images/courses/a/apple.webp`。
- 課程資料需要穩定 URL，未來可由 JSON 或 TypeScript 資料檔直接參照。
- 圖片不需要進入元件 import 流程，課程資料可以維持純資料。
- 未來 A-F 與 shared 圖片可用一致路徑擴充。

若採 `src/assets`，優點是可被 Vite 打包與 hash 管理；缺點是課程資料需要 import 或建置期映射，對目前資料模型較重。因此第一版先採 `public/assets`。

檔名規則：

- 課程專屬圖片：`a-[target]-[purpose].webp`
- 句子圖：`a-sentence-[short-slug].webp`
- 共用干擾圖：`shared-[target]-core.webp`
- 使用小寫英文字母、數字與 hyphen。
- 不使用空白、中文、底線或版本號。
- 不把 `draft`、`final`、`new` 放入檔名。

## Alt 文字規則

- 使用繁體中文與台灣用語。
- 描述圖片中實際可見主體，不描述測驗答案或技術資訊。
- 句子圖可以描述主要情境，但要避免過長。
- 不加入「圖片」「插圖」等多餘字眼，除非語意需要。
- 不加入檔名、ID、路徑、asset status。

範例：

- `一顆帶著綠葉的紅蘋果`
- `一隻正在搬食物的小螞蟻`
- `一隻張開嘴巴的友善短吻鱷`

## 核心單字圖片規格

### apple

- 圖片內容：一顆紅蘋果。
- 可包含：一片綠葉。
- 不可包含：多顆蘋果、人物、文字、品牌貼紙、商品包裝。
- 構圖：主體置中，完整顯示，不切到主體。
- 建議 alt：`一顆帶著綠葉的紅蘋果`
- 建議檔名：`a-apple-core.webp`

### ant

- 圖片內容：一隻小螞蟻正在搬一小塊食物。
- 風格：卡通化、友善、圓潤，不寫實。
- 不可包含：螞蟻群、昆蟲寫實細節、恐怖或密集畫面。
- 構圖：主體清楚，食物塊可見，背景簡潔。
- 建議 alt：`一隻正在搬食物的小螞蟻`
- 建議檔名：`a-ant-core.webp`

### alligator

- 圖片內容：一隻友善的綠色短吻鱷。
- 嘴巴：微微張開，但不露出過度尖銳或恐怖牙齒。
- 可包含：簡單河岸或草地。
- 不可包含：追逐、攻擊、張牙舞爪、寫實皮膚紋理。
- 建議 alt：`一隻張開嘴巴的友善短吻鱷`
- 建議檔名：`a-alligator-core.webp`

## 句子情境圖片規格

### An apple.

- 需求：單一蘋果，可放在桌面或野餐墊上。
- 不加入人物。
- 強調單一物件。
- 不出現文字。
- 建議檔名：`a-sentence-an-apple.webp`

### I see an ant.

- 需求：一位兒童蹲下觀察螞蟻，螞蟻仍需看得見。
- 場景簡單，人物表情自然友善。
- 不出現文字。
- 避免畫面太複雜。
- 建議檔名：`a-sentence-see-ant.webp`

### The alligator is big.

- 需求：一隻體型明顯較大的友善短吻鱷。
- 可搭配小石頭、草叢或小鳥呈現比例。
- 不呈現追逐或攻擊。
- 畫面仍適合兒童。
- 建議檔名：`a-sentence-big-alligator.webp`

## 圖片驗收標準

### 內容

- 主體正確。
- 無錯誤物件。
- 無多餘主體。
- 無文字。
- 無水印。
- 無商標或品牌包裝。

### 教學

- 學生能快速辨識。
- 不需猜測目標物件。
- 不會和其他選項混淆。
- 符合 A 課程語意。
- 干擾圖不因品質或背景差異洩漏答案。

### 構圖

- 主體完整。
- 主體置中。
- 邊界足夠。
- 不被裁切。
- 適合卡片裁切。

### 風格

- 與其他 A 圖片一致。
- 兒童友善。
- 不寫實。
- 不恐怖。
- 色彩清楚。
- alligator 不具威脅感。
- ant 不呈現過度寫實昆蟲細節。

### 技術

- 比例正確。
- 尺寸正確。
- 格式為 WebP。
- 檔名正確。
- 檔案大小合理。
- alt 已建立。
- manifest 狀態不可直接標示 `approved`。

## 後續圖片生成規則

1. 同一專案的所有圖片在同一個 ChatGPT 對話中連續生成。
2. 不為每張圖另開對話。
3. 每次生成前重申既有風格。
4. 核心單字圖先生成並審核。
5. 核心單字圖通過後，才生成句子情境圖。
6. 句子圖需沿用核心單字外觀。
7. 干擾圖需維持相同品質與複雜度。
8. 每張圖生成後先驗收，不直接全部放入網站。
9. 未通過圖片不得標示 `approved`。
10. 未經確認不覆蓋原圖。
