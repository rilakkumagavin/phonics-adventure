# Phase 12 實機與人工流程驗收紀錄

更新日期：2026-07-06

## 使用方式

本文件用來記錄 Phase 12 尚需真人與實機確認的項目。填寫前請先確認：

- 不將學生姓名、錄音或個人資料寫入本文件。
- 不把未測項目標成通過。
- 不以桌機瀏覽器結果取代 iPad Safari 或 Android Chrome 實機結果。
- 若任一核心流程失敗，Phase 12 仍維持未完成。

結果欄請使用：

- `通過`：已在指定裝置實際操作，符合預期。
- `需修正`：操作失敗、畫面破版、音訊或錄音不可用。
- `需確認`：結果不穩定、裝置資訊不足，或需要第二次驗證。
- `未測`：尚未執行。

填寫過程中可用下列指令檢查目前狀態：

```powershell
npm.cmd run phase12:device-form:status
```

所有項目完成後，正式檢查：

```powershell
npm.cmd run phase12:device-form
```

正式檢查會確認必要狀態都不是 `未測`，且 iPad/Android 裝置資訊與錄音 MIME
類型都有填寫；它不會自動把任何項目標成通過。

## 1. 桌機完整人工流程

桌機測試網址：

```text
http://127.0.0.1:5173/
```

啟動桌機測試站：

```powershell
npm.cmd run dev
```

平板實機測試請勿使用 `127.0.0.1`，因為那只代表平板自己。請先在電腦執行：

```powershell
npm.cmd run phase12:device-server:dry-run
```

確認列出的 LAN URL 後，正式啟動：

```powershell
npm.cmd run phase12:device-server
```

iPad Safari 與 Android Chrome 需與此電腦在同一個 Wi-Fi / LAN，並使用腳本列出的
`http://<LAN-IP>:5173/` 作為測試網址。

## 完成條件

Phase 12 只能在下列條件全部成立後才可宣告完成：

- 本文件所有必要項目皆填為 `通過`。
- `docs/acceptance/audio-listening-results.md` 的 42 個音訊皆有人工結果。
- 所有 `需修正` 或 `需重錄` 項目已處理並重新驗收。
- 自動測試、lint、型別檢查、建置與格式檢查仍通過。

## 仍不可做

- 不因建立本文件而修改音訊 manifest 的 `reviewStatus`。
- 不宣稱 iPad Safari 或 Android Chrome 已通過，除非本文件已有實機證據。
- 不上傳學生錄音。
- 不新增後端、登入、正式資料庫或外部 API。
- 不擴大到 G～Z。
