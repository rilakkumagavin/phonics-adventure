# Phonics Adventure

一個為英語初學者設計的遊戲化自然發音學習網站。

本專案以「聽、說、讀、每日複習」為核心，透過短時間、反覆出現、不同形式的遊戲任務，協助學習者建立字母、聲音、單字與簡短句子之間的連結。

---

目前專案已進入 MVP 整合驗收階段。

已完成：

* 建立 A～F 正式課程資料、圖片規格與圖片資產
* 建立音訊播放、慢速播放與防止重疊播放
* 建立可依課程活動載入 A～F 資料的聽音選圖、字母圖片配對、錄音回放、聲音分類與看字選答案
* 建立只接受有效 A～F 課程活動、需完成聽說讀才標記字母完成、可記錄答對與錯誤嘗試，並能安全拒絕損壞資料與處理儲存失敗的本機學習進度

# 1. 專案背景

英語初學者在學習自然發音時，經常會遇到以下問題：

* 一次接觸太多字母與規則
* 看過影片後很快忘記
* 能跟著念，但無法自行辨識聲音
* 能認得單字，卻無法拼讀新字
* 練習偏重看與背，缺少聽與說
* 每次練習內容完全相同，容易只記答案
* 缺少持續、短時間的每日複習

本網站不以「一次教完」或「一次記住」為目標。

學習者會透過多次接觸，逐漸完成：

1. 聽見聲音
2. 辨識聲音
3. 模仿聲音
4. 看見字母
5. 認讀單字
6. 閱讀短句
7. 在不同題型中再次遇見
8. 運用規則嘗試閱讀新字

---

# 2. 產品目標

本專案要建立一個讓英語初學者可以持續練習自然發音的學習網站。

網站應具備以下特性：

* 適合初學者
* 每次學習量少
* 每日學習時間短
* 可反覆播放聲音
* 可練習跟讀
* 可看見自己的學習進度
* 答錯時有分層提示
* 舊內容會依時間再次出現
* 每次複習不完全重複同一題
* 支援電腦與平板操作

---

# 3. 目標使用者

第一版主要服務：

* 英語初學者
* 國小低年級至中年級學生
* 尚未熟悉自然發音的學習者
* 需要反覆聽辨與跟讀的學生
* 適合由教師或家長陪同開始使用的學生

第一版不以成人進階英語學習者、考試準備者或完整文法課程為主要對象。

---

# 4. 核心學習能力

每個完整學習單元必須包含以下四個面向。

## 4.1 聽

學習者需要能夠：

* 聽字母音
* 聽單字
* 聽簡短句子
* 辨識單字開頭的聲音
* 比較不同聲音
* 從多個選項中找出正確聲音

---

## 4.2 說

學習者需要能夠：

* 聽原音
* 模仿字母音
* 跟讀單字
* 跟讀簡短句子
* 錄下自己的聲音
* 回放自己的聲音
* 比較原音與自己的發音
* 重新嘗試

第一版不宣稱能精準判斷發音正確率。

---

## 4.3 讀

學習者需要能夠：

* 辨識大寫與小寫字母
* 看見字母後回想聲音
* 看見單字後嘗試認讀
* 將聲音與字形連結
* 閱讀簡短句子
* 將已學規則運用到新字

---

## 4.4 每日複習

學習者不需要一次記住所有內容。

系統應依學習紀錄安排：

* 隔日複習
* 答錯項目優先複習
* 較不熟練能力優先出現
* 已熟練內容逐漸延長複習間隔
* 使用不同題型複習相同概念
* 控制每日任務數量

---

# 5. 教學影片的角色

自然發音影片可作為：

* 課程內容來源
* 字母與發音導入
* 教師備課參考
* 學生複習素材

影片不代表一個課程已經完成。

網站必須將影片內容拆成：

* 小型學習單位
* 聽音活動
* 發音活動
* 認讀活動
* 遊戲任務
* 每日複習題目

不得只嵌入影片後就視為完成教學功能。

# 8. 學習設計原則

## 8.1 每次學習量要小

每次新學習原則：

* 新聲音1～2個
* 新單字3～5個
* 新句子1～2句
* 學習時間約5～10分鐘

不得一次要求初學者記住大量字母與單字。

---

## 8.2 聽、說、讀要互相連結

單一字母課程不能只有：

* 看圖片
* 點選答案
* 背單字卡
* 看影片

每個完整單元都需要包含：

* 聽見聲音
* 嘗試說出聲音
* 看見字形
* 認讀單字
* 閱讀簡短句子
* 之後再次複習

---

## 8.3 共用遊戲，但不共用完全相同的內容模板

遊戲元件可以重複使用，但每個單字應依語意設計：

* 圖片情境
* 動作
* 例句
* 聽力任務
* 口說任務
* 閱讀任務
* 複習變化

例如 `apple` 與 `ant` 即使都以 A 開頭，也不應只有圖片和文字不同。

---

## 8.4 答錯時提供漸進式提示

學生答錯時，依序提供：

1. 再播放一次
2. 慢速播放
3. 顯示圖片、口型或發音提示
4. 減少選項
5. 顯示部分線索
6. 最後才顯示答案

不得只顯示紅色叉號或「錯誤」。

---

## 8.5 複習不能只重做相同題目

同一個聲音可以透過不同形式再次出現：

* 聽聲音選圖片
* 看圖片選字母
* 聽單字判斷開頭音
* 將圖片放入聲音分類區
* 跟讀單字
* 跟讀句子
* 看見新字後嘗試認讀

系統要練習相同能力，而不是讓學生記住固定答案位置。

# 9. 初步每日複習規則

第一版先採用簡化規則。

| 學習狀態  | 建議下次複習 |
| ----- | -----: |
| 第一次完成 |    1天後 |
| 再次答對  |    3天後 |
| 持續答對  |    7天後 |
| 穩定掌握  |   14天後 |
| 答錯    |    1天後 |
| 連續答錯  | 優先再次出現 |

每日任務內容建議：

* 60% 到期複習
* 20% 較弱能力
* 20% 新內容

實際規則會在每日複習引擎設計階段確認。

# 10. 隱私與資料原則

第一版不建立帳號與雲端資料庫。

學習紀錄原則：

* 儲存在目前裝置
* 不上傳伺服器
* 不蒐集姓名
* 不蒐集電子郵件
* 不追蹤跨網站行為
* 可讓使用者清除紀錄

錄音功能原則：

* 只在使用者主動按下錄音後啟動
* 明確顯示錄音狀態
* 不在背景錄音
* 預設不傳送到伺服器
* 可重新錄製
* 可刪除
* 麥克風權限被拒絕時，其他學習功能仍可使用
